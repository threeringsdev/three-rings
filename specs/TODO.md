## Decisions log

- 2026-07-13: **Auth shipped — path-B cookie proxy over Neon Auth (Better Auth), Google included; verification on.** The browser/webview only ever holds our origin's httpOnly cookies (`tr_session` upstream session, `tr_jwt` 15-min EdDSA JWT, `tr_challenge` OAuth window); the server proxies the auth REST API and re-mints JWTs silently. Google stays server-side via the verifier/challenge exchange (mechanism excavated from Neon's own SDK — the naive browser-side `/token` bridge dies on partitioned third-party cookies). Toggles (maintainer): `require_email_verification=true` (OTP method — code-in-app works identically in webviews) + Google (shared creds), both branches; that single toggle both gates sign-up sessions and auto-mails the OTP. Two latent bugs found: the step-5 verifier rejected every real token (jsonwebtoken v9 needs the expected `aud` configured; `iss`/`aud` = the auth origin), and a unit `Suspense` fallback (`|| ()`) desyncs leptos hydration app-wide (never ship `fallback=|| ()`). Tauri desktop spike PASS — the full gated flow works inside WKWebView against the embedded server; Google-in-webview is impossible (Google policy + upstream callback rules), so desktop Google = system browser + loopback callback, queued as a Phase 3 task. `SET LOCAL app.user_id` still rides the data-model per-request transaction.
- 2026-07-12: **Dockerfile = cargo-chef dep caching; Render build cache verified persistent.** The delivery Dockerfile now cooks dependencies in layers keyed only on the manifests/lockfile (two cooks matching cargo-leptos 0.3.7's real invocations: native `server-release`, wasm `release` into `target/front`), so code-only pushes reuse them — local Docker measured cold 137s → warm 9s. Frontend tools (tailwindcss v4.2.1, wasm-opt/binaryen version_123, wasm-bindgen 0.2.103) are pinned onto PATH in the image because cargo-leptos prefers PATH tools over downloading — removes per-build GitHub downloads *and* the unpinned "latest" API checks (rate-limit risk on shared builder IPs); the wasm-bindgen ARG must track the Cargo.toml `=0.2.103` pin. mold added for native links (scoped via `$CARGO_HOME/config.toml`; identical flags for cook + build keep fingerprints stable). The spec's Render-cache caveat is resolved: deploy build logs show a registry cache-manifest import with `CACHED` build-stage layers on an ephemeral builder → persistence confirmed, sccache+S3 stays the fallback if cook layers run cold (follow-up task queued).
